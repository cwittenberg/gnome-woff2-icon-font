How to Use This Font & Guidelines

Typography & Brand Colors

The visual identity of this page, including its typography and color palette, follows the official GNOME Brand guidelines. For comprehensive instructions on how to properly use these visual elements, spacing, and specific color hex codes in your materials, please refer to the GNOME Brand Book and visit brand.gnome.org.

The controls at the top of this page allow you to dynamically apply the exact GNOME HIG color values to the rendered font icons to test how they look.

Dynamic Loading Requirements

This visualization uses JavaScript's fetch() API to read your gnome.css file dynamically without hardcoding it. Because of browser security constraints regarding cross-origin requests, you must run this HTML file through a local web server (e.g., python -m http.server or NodeJS http-server). It will not work if simply double-clicked via the file:// protocol.

Embedding in Your Project

Ensure both gnome.css and gnome.woff2 are accessible. Link the CSS in your document head:

<link rel="stylesheet" href="gnome.css">

Then, render an icon by using an <i> tag with the base class and specific icon class:

<!-- Renders the desktop icon in HIG Blue -->
<i class="icon icon-desktop" style="color: #3584e4;"></i>
