How to Use This Font & Guidelines

Typography and Identity

The visual identity of this page and the materials for the GNOME project rely on consistent typography. For printed materials and websites, the Inter typeface should be used[cite: 3].

Applying GNOME Brand Colors

Color is a critical element of the GNOME visual identity and brand recognition[cite: 3]. You can style your icons using the official GNOME Human Interface Guidelines (HIG) color palette[cite: 3]. The color palette is categorized as follows[cite: 3]:

  • Primary: Blue (#3584e4)[cite: 3].
  • Secondary: Orange (#FF7800), Purple (#9141AC), Green (#33D17A), and Red (#E01B24)[cite: 3].
  • Neutral: White (#FFFFFF), Grey (#77767B), and Quite Black (#241F31)[cite: 3].

The controls at the top of this page allow you to dynamically apply these exact hex values to the rendered font icons using CSS variables.

Dynamic Loading Requirements

This visualization uses JavaScript's fetch() API to read your gnome.css file dynamically without hardcoding it. Because of browser security constraints regarding cross-origin requests, you must run this HTML file through a local web server (e.g., python -m http.server or NodeJS http-server). It will not work if simply double-clicked via the file:// protocol.

Embedding in Your Project

Ensure both gnome.css and gnome.woff2 are accessible. Link the CSS in your document head:

<link rel="stylesheet" href="gnome.css">

Then, render an icon by using an <i> tag with the base class and specific icon class:

<!-- Renders the desktop icon in HIG Blue -->
<i class="icon icon-desktop" style="color: #3584e4;"></i>
